Бэкэнд

Бэкэнд будет отвечать за взаимодействие с API SEWIO, обработку данных о местоположении и отправку предупреждений, когда сотрудник приближается к экскаватору ближе, чем на 5 метров.

Технологии

Node.js для создания серверной части.
Express.js для организации веб-сервера и обработки HTTP-запросов.
WebSocket (или библиотеки, такие как Socket.IO для Node.js), для обеспечения своевременной доставки предупреждений на фронт-энд.
Основные компоненты бэкэнда

Клиент API SEWIO: модуль для взаимодействия с API SEWIO для получения данных о местоположении меток.
Обработчик данных: компонент, который анализирует координаты меток, определяет их относительные расстояния и генерирует события при приближении сотрудника к экскаватору.
Менеджер событий: модуль для обработки сгенерированных событий и отправки уведомлений через WebSocket клиентам на фронт-энде.
Фронт-энд
Фронт-энд будет отвечать за отображение местоположений меток в реальном времени и уведомления о событиях.

Технологии

React для построения пользовательского интерфейса.
Redux или Context API для управления состоянием приложения.
WebSocket для получения событий от бэкэнда в реальном времени.
Основные компоненты фронт-энда

Карта местоположения: компонент для отображения текущих местоположений меток на карте или схеме территории.
Уведомления: модуль для отображения уведомлений о приближении сотрудников к экскаватору.
Панель управления: интерфейс для настройки параметров системы, например, для изменения критического расстояния для предупреждений.
Взаимодействие между компонентами
Клиент API SEWIO регулярно запрашивает данные о местоположении меток и передает их Обработчику данных.
Обработчик данных анализирует полученные данные, определяет расстояния между метками и, при необходимости, генерирует событие, которое передается Менеджеру событий.
Менеджер событий отправляет уведомления через WebSocket на фронт-энд, где они отображаются пользователю.
